title: Configu's Terminology
---

Here is a dictionary of some key concepts in Configu:

<div class="table-wrapper">

  <table>
    <thead>
      <tr>
        <th>Concept</th>
        <th>Explanation</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>
          <strong>Config</strong>
        </td>
        <td>
          A Config is a generic representation of a software configuration. It is the data model that Configu uses to store and fetch configuration data. A Config consists of four major properties:
          - key: The name of the configuration as it appears in the code and in the ConfigSchema.
          - schema: The unique identifier of the ConfigSchema that the configuration has been declared on.
          - value: The literal value of the configuration in the ConfigSet.
          - set: The path of the ConfigSet that the value belongs to.
          <a href="https://github.com/configu/configu/blob/main/types/Config.ts">See Config decleration.</a>
        </td>
      </tr>

      <tr>
        <td>
          <strong>Cfgu</strong>
        </td>
        <td>
          A Cfgu is a generic declaration of a Config, that specifies information about its type, constrains and other characteristics.
          <a href="https://github.com/configu/configu/blob/main/types/Cfgu.ts">See Cfgu decleration.</a>
        </td>
      </tr>

      <tr>
        <td>
          <strong>ConfigSchema</strong>
        </td>
        <td>
          A ConfigSchema is a file stored as \[Config.schema\].cfgu.\[json|yaml\] that contains binding records between a unique \[Config.key\] and its Cfgu declaration.
          <a href="/docs/config-schema">Read more on ConfigSchema</a>
          <a href="https://github.com/configu/configu/blob/main/types/ConfigSchema.ts">See ConfigSchema decleration.</a>
        </td>
      </tr>

      <tr>
        <td>
          <strong>ConfigSet</strong>
        </td>
        <td>
          A ConfigSet (\<Config.set\>) is a string representing a path in a hierarchical structure (tree) that is used to organize \<Config.value\>s for different contexts such as environments, tenants, etc.
          <a href="/docs/config-set">Read more on ConfigSet</a>
          <a href="https://github.com/configu/configu/blob/main/types/ConfigSet.ts">See ConfigSet decleration.</a>
        </td>
      </tr>

      <tr>
        <td>
          <strong>ConfigStore</strong>
        </td>
        <td>
          A ConfigStore is a storage engine for Configs, which can be any system or service that is capable of storing data. It can be a simple text file, a database, a secret manager, Configu's SaaS Configuration Store, or your own custom implementation.
          <a href="/docs/config-store">Read more on ConfigStore</a>
          <a href="https://github.com/configu/configu/blob/main/types/ConfigStore.ts">See ConfigStore decleration.</a>
        </td>
      </tr>

      <tr>
        <td>
          <strong>Configuration file</strong>
        </td>
        <td>
          In computing, configuration files are files used to configure the parameters and initial settings for some computer programs. They are used for user applications, server processes and operating system settings.
          <a href="https://en.wikipedia.org/wiki/Configuration_file">Read more on Configuration file</a>
        </td>
      </tr>

      <tr>
        <td>
          <strong>Configuration formats</strong>
        </td>
        <td>
          XML, JSON, YAML, TOML, Dotenv, HCL, etc..
        </td>
      </tr>

      <tr>
        <td>
          <strong>Configuration secret</strong>
        </td>
        <td>
          API keys, passwords, certificates, and other sensitive data.
        </td>
      </tr>

      <tr>
        <td>
          <strong>Configuration template</strong>
        </td>
        <td>
          A text that contains <a href="https://mustache.github.io/">
          \{\{mustache\}\}
          </a> templates and can be rendered as a configuration file.
        </td>
      </tr>

      <tr>
        <td>
          <strong>Configuration syncornizer</strong>
        </td>
        <td>
          A process that syncs configuration data through Configu to a runtime environment such as Vercel, Netlify, AWS Lambda, AWS ECS, K8s, etc.
        </td>
      </tr>
    </tbody>

  </table>

</div>
